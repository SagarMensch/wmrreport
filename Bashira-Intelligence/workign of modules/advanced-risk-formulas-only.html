1. 3D Risk Topography

State-Space Risk Function

Let s = operational state vector
s = [progress_gap, progress_velocity, schedule_variance,
     queue_exposure, dependency_pressure, anomaly_score,
     rig_delay, resource_conflict, stage_stall]
Risk(s) = f(s)
Example linearized approximation:
Risk = w1*progress_gap
     + w2*(-progress_velocity)
     + w3*schedule_variance
     + w4*queue_exposure
     + w5*dependency_pressure
     + w6*anomaly_score
     + w7*rig_delay
     + w8*resource_conflict
     + w9*stage_stall
Bounded score form:
Risk_surface = 100 * sigmoid(Risk)

The surface chart visualizes Risk over a projected 2D state manifold with height = risk intensity.

2. EVT Tail Risk

Tail Quantile

Let L be loss / exposure random variable
P99(L) = inf { x : F_L(x) >= 0.99 }
P95(L) = inf { x : F_L(x) >= 0.95 }

Generalized Pareto Tail Approximation

For threshold u and exceedance y = L - u,  y > 0
P(Y <= y | Y > 0) = 1 - (1 + xi*y/beta)^(-1/xi)
Tail exposure estimate:
EVT_P99 = u + (beta/xi) * ((N/N_u * (1 - 0.99))^(-xi) - 1)

Expected Shortfall

ES_q = E[L | L > VaR_q]

Useful when average tail damage matters more than just the percentile cutoff.

3. NHPP Event Engine

Intensity Function

lambda(t) = instantaneous event arrival rate
NHPP count process:
N(t) ~ Poisson( Lambda(t) )
Lambda(t) = integral from 0 to t of lambda(u) du

Stage-Sensitive Intensity

lambda(t) = a0
          + a1*complexity(t)
          + a2*dependency_load(t)
          + a3*resource_conflict(t)
          + a4*queue_pressure(t)

Event Sampling

delay_events_t ~ Poisson(lambda(t) * delta_t)

If lambda rises through the program, disruption arrivals accelerate rather than remaining constant.

4. Monte Carlo Delay Simulation

Path Simulation

For simulation k = 1 ... K:
  Initialize state s_0 from latest snapshot
  For each step t:
    Sample event arrivals from NHPP
    Sample duration overruns
    Sample dependency failures
    Update graph states and schedule states
    Accumulate delay D_k and cost C_k

Distribution Outputs

Expected Delay = (1/K) * sum(D_k)
P95 Delay = empirical 95th percentile of {D_k}
P99 Delay = empirical 99th percentile of {D_k}
Expected Cost = (1/K) * sum(C_k)

5. Execution Price / Financial Exposure

Base Delay Cost

Execution_Risk_Cost = Expected_Delay * Daily_Cost_of_Delay

Expanded Enterprise Pricing

Total_Cost = (Delay_Days * Burn_Rate)
           + Rig_Standby_Cost
           + Crew_Idle_Cost
           + Resequencing_Penalty
           + Dependency_Spillover_Cost
           + Tail_Risk_Surcharge

Heavy Risk Pricing

If state = critical:
  Adjusted_Cost = Total_Cost * m_critical
Example:
m_critical = 2.5

6. Classification Layer

Risk State Mapping

signal(x) =
  DONE      if progress >= 100 or residual_risk <= tau_done
  DELAYED   if tau_done < risk < tau_critical
  CRITICAL  if risk >= tau_critical or contagion >= tau_contagion

Maximum Delay Probability

p_delay(i) = P( D_i > 0 | current_state_i )
Max_Delay_Prob = max_i p_delay(i)

Recovery Confidence

Recovery_Confidence = 100 - g(risk, queue_exposure, anomaly_score,
                              data_gap, rig_delay, dependency_pressure)

7. Graph and Propagation Logic

Execution Graph

G = (V, E)
V = {wells, rigs, crews, tasks, clusters, corridors}
E = dependency / adjacency / shared-resource links

Node Update

h_i^(t+1) = phi( h_i^(t), aggregate_{j in N(i)}( e_ij, h_j^(t) ), x_i^(t) )

Generic graph message-passing form.

Propagation Score

Propagation(i) = sum_{j in N(i)} w_ij * FailureSignal(j)
Cluster_Risk(c) = sum_{i in c} Risk(i) + alpha * Spillover(c)

Lead Rig Centrality

Lead_Rig = argmax_r Centrality(r)
Centrality(r) may be:
  degree(r)
  betweenness(r)
  propagated_loss_contribution(r)
  downstream_delay_impact(r)

8. ST-GNN Formulation

Spatio-Temporal Update

H^(t+1) = F_graph( A, H^(t), X^(t) ) + F_time( H^(t-k:t) )
Node risk at horizon h:
Risk_i^(t+h) = psi( H_i^(t+h) )

Sequence Failure Propagation

FailureSpread_i = sum_{j in N(i)} A_ij * Delay_j * Sensitivity_ij
Updated_Risk_i = Base_Risk_i + beta * FailureSpread_i

9. MARL Objective

Multi-Agent Policy

Agents = {rigs, crews, dispatchers, planners, clusters}
a_t^m = action of agent m at time t

Reward Function

R_t = - c1*delay_growth
      - c2*tail_exposure
      - c3*queue_contagion
      - c4*idle_resources
      - c5*critical_wells
      + c6*throughput
      + c7*recovery_probability

Optimal Policy

pi* = argmax_pi E_pi [ sum_t gamma^t R_t ]

A suggested reroute is an action extracted from a policy that improves expected portfolio reward.
